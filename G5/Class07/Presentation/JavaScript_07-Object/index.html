Introduction

  • Objects group together a set of variables and functions to create a model of a something you would recognize from the real world
  • In an object:
    • Variables become know as properties (Tell us more about the object)
    • Functions become know as methods (Tasks that are associated with the object or what the object can do)

Introduction (2)

  • Like variables and named functions, properties and methods have a name and a value. In an object, that name is called a key
  • An object cannot have two keys with the same name
  • The value of a property can be a string, number, Boolean, array, or even another object. The value of a method is always a function

The Object Oriented Paradigm

Note: This section is only introductory

Don't panic

Object-Oriented Programming

  • Objects interacting with one another through methods and properties
  • Used to store data, structure applications into modules and keeping code clean
  • A widely accepted definition to classify a programming language such as Object Oriented is based on two requirements: its capability to model a problem through objects and its support of some principles that grant modularity and code reuse.

JavaScript OOP versus classical OOP

  • Classical Object-Oriented programming has two types of abstractions: classes and objects.
    • An object is an abstraction of a real-world entity, can only be created by instantiating classes
    • A class is an abstraction of an object or another class (in other words, it is a generalization)
  • JavaScript has a different approach to object management, has just one type of abstraction - the objects. An object can be created directly as an abstraction of a real-world entity or as an abstraction of another object (prototype).
  • As opposed to the classical OOP approach, the JavaScript approach is sometimes called Prototypal Object-Oriented Programming

JavaScript OOP versus classical OOP (2)

  • In classical OOP inheritance is an operation allowed on classes, in prototypal OOP inheritance is an operation on objects
  • That does not mean that classical OOP is better than prototypal OOP or vice versa. They are simply different approaches.
  • In classical OOP classes are immutable, that is we cannot add, change, or remove properties or methods at runtime, in prototypal OOP objects and prototypes are extremely flexible
  • Classical OOP adds an extra level of abstraction with classes, leading to a more verbose code, while prototypal OOP is more immediate and requires a more compact code

Creating Objects

Literal Notation

  • Easiest and most popular method for creating objects
  • The obect is the curly braces and their content
  • Separate each key from its values by using a colon
  • Separate each property and methos with comma (not after the last value)


let hotel = { 
  name : "Quay",
  rooms: 40,
  booked: 25,
  gym: true,
  roomTypes: ['twin', 'double', 'suite'],

  checkAvailability: function(){
    return this.rooms - this.booked;
  }
};

Accessing an Object

  • Properties and methods of an object can be accessed using the dot notation
  • You can also access properties using the square brackets
  • Dot Notation: Name of the object, followed by a period, then the name of the property or method you want to access
  • The period is know as member operator

Constructor Notation

  • The new keyword and the Object() constructor create a blank object. You can then add properties and methods to the object using the dot notation
  • Each statement that adds a property or a method should end in semicolon

Note

  • You can use the dot syntax to add properties and methods to any object you have created (no matter which notation you used to create it)
  • To create an empty object using the literal notation use: let hotel = {}

Updating Objects

  • To update the value of properties, use dot notation or square brackets (no matter which notation you used to create the object)
  • If property does not exist, it will be added to the object
  • To delete a property, use the delete followed by the object name and the property name: 

    
							delete hotel.name;
